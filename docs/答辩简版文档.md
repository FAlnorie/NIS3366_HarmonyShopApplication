# 基于 HarmonyOS 的商城应用

## 答辩简版文档

---

## 一、项目概述

本项目是一个基于 HarmonyOS Next + ArkTS 开发的商城应用前端系统，围绕“用户登录后完成商品浏览、加购与结算演示”的核心流程实现。

项目定位：课程实践项目，强调功能闭环、工程结构和可演示性，不接真实后端与支付。

---

## 二、项目目标

1. 完成一个可运行的商城前端业务闭环。
2. 实现用户系统（注册、登录、持久化、访问控制）。
3. 实现商品浏览、详情、购物车、结算演示、用户中心。
4. 通过模块化设计提升代码可维护性与可扩展性。

---

## 三、核心功能

### 1. 用户与认证

- 注册：用户名去重、密码规则校验。
- 登录：用户名密码验证。
- 持久化：用户数据与当前登录态本地保存。
- 访问控制：未登录无法进入业务页面。

### 2. 商品浏览

- 首页商品双列展示。
- 分类切换（精选/手机/服饰/电脑/配件等）。
- 下拉刷新与列表滚动。
- 点击商品进入详情页。

### 3. 购物车

- 详情页加入购物车。
- 查看购物车列表。
- 数量增减、删除单项、清空购物车。
- 动态计算合计金额。

### 4. 结算演示

- 确认订单页展示：收货信息、商品清单、支付方式、配送方式、金额明细。
- 提交订单演示：提交中 -> 成功提示（无真实支付）。

### 5. 用户中心

- 展示当前账号信息。
- 展示购物车件数与总额。
- 提供查看购物车、清空购物车、退出登录。

---

## 四、系统设计简述

### 1. 分层结构

```text
页面层: Login / Register / Index / ProductDetail / Cart / Checkout / UserCenter
组件层: CategoryTab / ProductCard / ProductItemList
数据层: ProductData / CartStore / AuthStore
```

### 2. 关键模块职责

- `AuthStore`：注册、登录、登录态校验、用户数据持久化。
- `CartStore`：购物车增删改查与金额计算。
- `ProductData`：商品模拟数据与按分类组织。

---

## 五、关键技术点

1. ArkTS 声明式 UI 与组件化开发。
2. 路由跳转与页面间参数传递（商品详情）。
3. 响应式状态管理（`@State` / `@Prop` / `@Link`）。
4. 本地持久化（`PersistentStorage + AppStorage`）。
5. 页面级登录守卫（未登录自动跳转登录页）。

---

## 六、演示流程（答辩建议）

1. 启动应用 -> 登录页。
2. 新用户注册 -> 返回登录。
3. 登录成功 -> 首页浏览商品。
4. 点击商品 -> 进入详情 -> 加入购物车。
5. 进入购物车 -> 调整数量 -> 去结算。
6. 在结算页提交订单（演示成功提示）。
7. 进入用户中心 -> 查看数据 -> 退出登录。

---

## 七、项目亮点

1. 业务闭环完整：从登录到下单演示可全流程跑通。
2. 文档、页面、数据结构一致性较好。
3. 用户功能与访问控制已落地，不是纯展示型列表项目。
4. 模块边界清晰，便于继续扩展后端能力。

---

## 八、当前不足

1. 未接入真实后端与数据库。
2. 密码未进行安全加密与传输保护（当前为本地演示）。
3. 购物车/订单数据尚未实现服务端同步。
4. 结算与支付为前端模拟流程。
