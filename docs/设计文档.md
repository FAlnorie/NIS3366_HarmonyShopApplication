# 基于 HarmonyOS 的商城应用

## 设计文档

---

## 一、设计目标

当前版本设计目标是构建一个可完整演示核心电商流程的 ArkTS 前端应用：

- 账号注册与登录
- 登录态访问控制
- 商品列表与详情浏览
- 购物车管理
- 结算页提交流程演示
- 用户中心信息与快捷操作

系统强调模块化、可读性和课程项目可演示性，不依赖后端服务。

---

## 二、系统架构设计

### 2.1 分层架构

```text
页面层（Pages）
  Login / Register / Index / ProductDetail / Cart / Checkout / UserCenter
      ↓
组件层（Components）
  CategoryTab / ProductCard / ProductItemList
      ↓
数据与状态层（Data & Store）
  ProductData / CartStore / AuthStore
```

### 2.2 架构说明

1. 页面层：负责页面路由、业务流程和交互编排。
2. 组件层：负责复用 UI 片段，降低页面重复代码。
3. 数据与状态层：
- `ProductData` 提供商品静态数据。
- `CartStore` 管理购物车内存数据。
- `AuthStore` 管理用户数据与登录态（本地持久化）。

---

## 三、功能模块设计

### 3.1 认证模块（AuthStore + Login/Register）

#### 3.1.1 数据结构

```ts
class UserAccount {
  username: string
  password: string
  nickname: string
}
```

#### 3.1.2 关键能力

- `register(username, password, nickname)`：注册与校验。
- `login(username, password)`：登录校验。
- `logout()`：清理当前登录态。
- `isLoggedIn()`：登录状态判断。
- `getCurrentUser()`：获取当前用户信息。

#### 3.1.3 持久化方案

- 通过 `PersistentStorage + AppStorage` 保存：
- 用户列表 JSON
- 当前登录用户名

#### 3.1.4 页面行为

- `Login`：登录成功 `replaceUrl` 到首页。
- `Register`：注册成功返回登录页。

---

### 3.2 访问控制模块

在 `Index / ProductDetail / Cart / Checkout / UserCenter` 页面的生命周期中执行登录校验：

1. 进入页面。
2. 调用 `AuthStore.isLoggedIn()`。
3. 未登录则 `router.replaceUrl('pages/Login')`。

该设计实现了“登录后可访问业务页面”的约束。

---

### 3.3 商品模块

#### 3.3.1 商品列表

- `Index` 页面承载商品主视图。
- `CategoryTab` 提供分类切换入口。
- `ProductItemList` 使用 `Refresh + List + LazyForEach` 渲染商品。
- `ProductCard` 负责单个商品卡片展示。

#### 3.3.2 商品详情

- 点击商品卡片跳转 `ProductDetail`。
- 通过路由参数 `productId` 查询并展示商品信息。
- 支持“加入购物车”。

---

### 3.4 购物车模块（CartStore + Cart）

#### 3.4.1 数据结构

```ts
interface CartItem {
  product: Product
  quantity: number
}
```

#### 3.4.2 核心接口

- `getItems()`
- `addItem(product, quantity)`
- `updateQuantity(productId, quantity)`
- `removeItem(productId)`
- `clear()`
- `getTotalCount()`
- `getTotalPrice()`

#### 3.4.3 页面能力

- 显示商品项与数量。
- 支持 `+ / - / 删除 / 清空`。
- 动态计算并展示合计金额。

---

### 3.5 结算演示模块（Checkout）

#### 3.5.1 页面内容

- 收货信息区
- 商品清单区
- 支付方式选择
- 配送方式选择
- 金额明细与应付金额

#### 3.5.2 提交流程（演示）

1. 点击提交订单。
2. 进入“提交中”状态。
3. 延时后展示“提交成功”反馈。
4. 不对接真实支付或后端订单系统。

---

### 3.6 用户中心模块（UserCenter）

#### 3.6.1 展示信息

- 当前用户昵称与账号。
- 购物车商品数量与总额。

#### 3.6.2 快捷操作

- 查看购物车
- 清空购物车
- 退出登录

---

## 四、路由与页面设计

### 4.1 页面清单

- `pages/Login`
- `pages/Register`
- `pages/Index`
- `pages/UserCenter`
- `pages/ProductDetail`
- `pages/Cart`
- `pages/Checkout`

### 4.2 启动流程

- `EntryAbility` 启动加载 `pages/Login`。
- 登录后进入首页。

---

## 五、关键交互流程

### 5.1 注册与登录

```text
Register -> 输入并校验 -> 注册成功 -> Login -> 登录成功 -> Index
```

### 5.2 浏览与加购

```text
Index -> 点击商品 -> ProductDetail -> 加入购物车 -> Cart
```

### 5.3 结算演示

```text
Cart -> 去结算 -> Checkout -> 提交订单(演示) -> 成功提示
```

### 5.4 用户中心

```text
Index 顶部按钮 -> UserCenter -> 查看购物车/清空购物车/退出登录
```

---

## 六、数据设计

### 6.1 商品数据

- 来源：`ProductData`
- 组织方式：按分类的 `Record<string, Product[]>`

### 6.2 用户数据

- 来源：`AuthStore`
- 组织方式：JSON 字符串持久化存储

### 6.3 购物车数据

- 来源：`CartStore`
- 当前实现：内存态（进程内）

---

## 七、异常与边界处理

1. 登录失败：提示用户名或密码错误。
2. 注册失败：提示用户名重复/密码不合法/确认密码不一致。
3. 详情页参数异常：展示“未找到商品”。
4. 空购物车：显示空状态文案与返回入口。
5. 未登录访问业务页：统一重定向到登录页。

---

## 八、可扩展性设计

当前设计保留以下扩展空间：

1. `AuthStore` 接入后端鉴权与密码加密。
2. `CartStore` 改为持久化或服务端同步。
3. `Checkout` 接入真实订单与支付 API。
4. 增加搜索、收藏、订单列表等功能模块。

---

1. 认证流程：注册、重复注册、错误登录、退出登录。

3. 购物车流程：加购、增减、删除、清空、金额计算。
