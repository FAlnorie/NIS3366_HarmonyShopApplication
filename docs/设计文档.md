
# 基于 HarmonyOS 的商城应用

## 设计文档

---

## 一、总体设计

### 1.1 设计目标

本系统基于 HarmonyOS Next 开发，采用 ArkTS 声明式开发模式，实现一个支持：

* 商品列表展示
* 列表滑动
* 下拉刷新
* 懒加载
* 多品类切换

的简易商城应用。

设计遵循：

* 模块化
* 低耦合
* 高内聚
* 易扩展
* 易维护

---

## 二、系统架构设计

### 2.1 总体架构

本系统采用典型的前端分层结构：

```
UI层（页面展示层）
   ↓
状态管理层（数据状态控制）
   ↓
数据层（模拟数据源）
```

### 2.2 架构说明

#### （1）UI 层

负责页面渲染与用户交互，包括：

* 商品列表界面
* 品类切换组件
* 下拉刷新交互
* 到底提示展示

使用组件：

* List
* Scroll
* LazyForEach
* Tabs（或自定义按钮）


#### （2）状态管理层

负责：

* 商品列表数据
* 当前品类状态
* 是否正在刷新

采用 ArkTS 响应式变量进行状态管理。

#### （3）数据层

* 使用本地模拟数据
* 按品类分类存储

## 三、模块设计

### 3.1 商品卡片模块设计

#### 功能：

展示单个商品项数据。

#### 实现方式：

接受外部传入的 product 对象展示数据，实现组件复用。布局采用Column容器，自上而下排列图片、名称和价格文本。

#### 商品项布局设计：

```
--------------------
|     商品图片      |
--------------------
|     商品名称      |
| 价格        购买  |
--------------------
```

---

### 3.2 品类切换模块设计

#### 功能：

支持精选、手机、服饰等品类切换。

#### 实现方式：

使用 Tabs 或 Row + Button 实现顶部分类栏。在用户点击品类的时候触发 onClick 数据更新状态变量并传出。

#### 状态变量：

```
currentCategory: string
```

#### 切换流程：

1. 点击品类
2. 更新 currentCategory
3. 商品列表模块接收外部传入的新数据
4. 清空当前数据
5. 加载该品类数据

---

### 3.3 商品列表模块设计

#### 功能：

支持上下滚动展示、懒加载商品项，下滑重新加载当前品类数据，滑动到底部出现到底提示。

#### 实现方式：

接受外部传入的 product list 数据，将其中的 product 对象传入商品卡片模块并分成两列进行展示。使用 ArkTS 的 Refresh 实现刷新，使用 List 组件实现上下滚动，在 List 底部追加提示组件实现到底提示。

#### 结构设计：

```
Refresh
 └── List
      ├── LazyForEach
      |     └── ListItem
      |           ├── Image
      |           ├── Text（商品名称）
      |           └── Text（商品价格）
      └── ListItem（Bottom Tip）
```

#### 滑动刷新流程设计：

```
用户下拉
   ↓
触发 onRefreshing
   ↓
isRefreshing = true
   ↓
重新加载数据
   ↓
isRefreshing = false
```
